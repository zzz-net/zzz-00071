# 维修备件借还管理系统

本地桌面工具，使用 Python + Tkinter + SQLite 构建，实现维修备件目录、库存、借用、归还、审批、追溯和导出等完整功能。

## 功能特性

- **备件管理**：新增、编辑、停用备件，配置规格、单价、审批阈值
- **库存管理**：实时库存展示，支持手动库存调整，完整变动追溯
- **借用流程**：提交借用 → 自动判断是否需审批 → 主管审批 → 借出 → 归还
- **审批管理**：高价值/受限备件需主管审批，支持通过和驳回
- **状态区分**：可借、已借空、待审批、已借出、已归还、已驳回、已回滚、已撤销
- **异常处理**：借出超库存、绕过审批、归还无记录等失败路径均有保护
- **回滚/撤销**：异常回滚恢复库存，撤销待审批申请
- **历史追溯**：库存变动历史、操作日志完整记录，可按备件过滤
- **数据导出**：CSV 导出库存明细、借还记录、库存变动历史，字段与界面一致
- **本地持久化**：SQLite 数据库存储，重启后所有数据（审批、备注、回滚痕迹、导出结果）保持一致

## 环境要求

- Python 3.7+（Windows 自带或官方安装包均可）
- 无需额外第三方依赖（Tkinter、sqlite3、csv 均为 Python 标准库）

## 启动方式

```bash

```

## 功能验证样例

### 样例 1：普通备件借用 → 归还（无需审批）

1. 以 **张三**（operator）登录
2. 在【备件库存】页选择 "SP-003 机械硬盘1TB"（可用30块）
3. 点击【提交借用】，数量 2，用途 "维修工作站"，确定
4. 观察：
   - 该备件可用库存变为 28
   - 【借还记录】页新增一条状态为"已借出"的记录
   - 【库存变动历史】可看到 -2 的变动记录
5. 同用户在【借还记录】选中该记录，点击【归还】，数量 2，备注 "完好归还"
6. 观察：库存恢复为 30，记录状态变为"已归还"

### 样例 2：高价值备件借用 → 主管审批 → 部分归还

1. 以 **张三** 登录
2. 借用 "SP-001 CPU处理器" x2（单价2200，超阈值需审批）
3. 观察：【借还记录】状态为"待审批"，【审批管理】页可看到该申请
4. 切换用户（右上角）以 **王主管** 登录
5. 在【审批管理】页选中记录，点击【审批通过】，填写备注
6. 观察：CPU 可用库存 10→8，记录状态"已借出"
7. 归还 1 片，备注"维修完成，剩余1片继续使用"
8. 观察：可用库存 8→9，记录状态仍为"已借出"，已归还数量=1

### 样例 3：失败路径 — 借出数量超过库存

1. 任何用户登录

5. 检查【库存变动历史】和【操作日志】：库存无变化，操作日志记录一条失败操作

## 界面说明

系统提供 5 个标签页：

1. **备件库存**：备件列表（状态颜色区分：绿色=可借、黄色=待审批、红色=已借空），支持搜索、分类过滤、新增/编辑/停用/调整/借用/导出
2. **借还记录**：所有借还记录，按状态颜色标记，支持状态筛选、搜索、归还、回滚、撤销、导出
3. **审批管理**：仅主管用户可操作，列出所有待审批申请，支持审批通过/驳回
4. **库存变动历史**：所有库存增减变动的完整流水，支持按备件过滤、导出
5. **操作日志**：系统所有操作记录（含失败操作），用于审计追溯

## 数据一致性保障

- 所有库存变更操作使用数据库事务，失败自动回滚
- 库存变动前校验库存充足性，失败不写入库存变动日志
- 失败操作仍写入操作日志（`success=0`），便于审计
- 状态流转采用 CHECK 约束，非法状态无法写入数据库
- 外键约束确保引用完整性（借还记录必须关联存在的备件和用户）
